# Анализ A/B‑теста премиум‑подписки в дейтинговом сервисе

## Основная цель:
Оценить успешность эксперимента по повышению стоимости премиум‑подписки (premium_no_trial) для новых пользователей, подключённых через две новые платёжные системы. Проверить, привело ли изменение к:
* росту средней выручки на платящего пользователя (ARPPU) в категории premium_no_trial;
* общему росту ARPPU по всему приложению (за счёт «оттока» пользователей в другие продукты).

## Методы достижения цели:
1. Разведочный анализ данных (EDA):
* проверка корректности сплитования (распределение по странам, возрасту, полу);
* анализ транзакций (сроки, платёжные системы, средний чек);
* визуализация ключевых метрик.
2. Статистический анализ:
* сравнение конверсии в премиального клиента между группами с помощью теста Хи‑квадрат (с поправкой Бонферрони на множественные сравнения);
* оценка ARPPU для premium_no_trial и всего приложения;
* применение бутстрапа для построения доверительных интервалов (из‑за ненормальности распределения и малого размера выборки).
3. Проверка гипотез:
* Гипотеза 1: повышение цены → рост ARPPU в premium_no_trial.
* Гипотеза 2: повышение цены → рост общего ARPPU (за счёт переключения на другие продукты).

## Используемый стек:
Python | Pandas | Numpy | Scipy | Matplotlib | Seaborn | Requests | Urllib

## Полученные результаты:
1. Конверсия в премиального клиента в тестовой группе ниже, чем в обеих контрольных (статистически значимо по тесту Хи‑квадрат).
2. ARPPU для premium_no_trial в тестовой группе значительно выше, чем в контрольных (подтверждено бутстрапом).
3. Общий ARPPU (по всему приложению) в тестовой группе:
* выше, чем во 2‑й контрольной группе (статистически значимо);
* разница с 1‑й контрольной группой менее выражена.
4. Средний чек в тестовой группе превысил контрольные во всех категориях, кроме coins.
5. Отток пользователей в другие продукты не компенсировал падение конверсии полностью (вторая гипотеза подтверждена частично).

## Главный вывод:
Эксперимент **частично успешен**:
* Повышение цены подписки **достигло цели** в рамках категории premium_no_trial - ARPPU вырос.
* Однако **общий эффект для приложения неоднозначен**: рост общего ARPPU наблюдается только при сравнении с одной из контрольных групп.

## Рекомендации:

* увеличить размер выборки для снижения вариативности оценок;
* проанализировать причины низкого оттока пользователей в другие продукты;
* проверить возможные баги в данных (например, аномалии в датах регистрации/оплаты).
